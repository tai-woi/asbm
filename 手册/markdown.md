
简介
====

Markdown 是一种轻量级标记语言，使用易读易写的纯文本格式编写文档，通过指定的标记符号指示文字转换为特定的排版格式。它可以与 HTML 网页代码混编，可方便地导出 HTML、PDF 等格式。


标题
====

标题分为多行标题（两种，文字下方加`=`或`-`）和单行标题（六种，行首加多个`#`）。它们只是书写方式不同，实际显示效果是一样的。

注：在部分解析器中，标题会根据设置自动创建锚点。

```
多行标题一
====

多行标题二
----

# 单行标题一
## 单行标题二
### 单行标题三
#### 单行标题四
##### 单行标题五
###### 单行标题六
```

> 多行标题一
> ====
>
> 多行标题二
> ----
>
> # 单行标题一
> ## 单行标题二
> ### 单行标题三
> #### 单行标题四
> ##### 单行标题五
> ###### 单行标题六


段落
====

连续的几行显示在一个段落内，使用空行分段。

注：如果要强制换行而不分段（分段的间距较大），可以在行末使用[反斜线](#反斜线)，或者使用 HTML 换行代码`<br>`。

```
第一个段落，
还是第一个段落

反斜线在行末时，\
后面的文本换行符成为 HTML 换行符
```

> 第一个段落，
> 还是第一个段落
>
> 反斜线在行末时，\
> 后面的文本换行符成为 HTML 换行符


引用
====

在段落前面加`>`，或者在每一行前面加`>`，表示引用内容。引用可以嵌套。

```
> 引用的段落
>
> > 更深层次的引用
```

> 引用的段落
>
> > 更深层次的引用


列表
====

列表分为无序列表和有序列表两种。无序列表以`+`、`-`、`*`开头，有序列表以数字开头，后接一个`.`。起始符后面需要接至少一个空格或制表符。列表可以嵌套。

注：有序列表的序号是根据第一个序号的数字往后顺序排列的，后面的数字完全相同也无所谓。

```
无序列表：

+ 列表项目
   + 列表项目
   + 列表项目
+ 列表项目

有序列表：

2. 列表项目
   1. 列表项目
   1. 列表项目
2. 列表项目
```

> 无序列表：
>
> + 列表项目
>    + 列表项目
>    + 列表项目
> + 列表项目
>
> 有序列表：
>
> 2. 列表项目
>    1. 列表项目
>    1. 列表项目
> 2. 列表项目


任务列表
====

部分解析器支持任务列表。任务列表是一个列表，它的项目以`[ ]`（注意，中间有空格）表示未完成，`[x]`表示已完成。

```
1. [x] 任务一
1. [x] 任务二
1. [ ] 任务三
```

> 1. [x] 任务一
> 1. [x] 任务二
> 1. [ ] 任务三


表格
====

部分解析器支持表格。表格的列元素以`|`分割，并且第二行需要有以`-`表示的分隔符。可选的，用`:`放在左边，表示左对齐；或放右边，表示右对齐；或两边都放，表示居中对齐。

```
表头一|表头二|表头三|表头四
-----|:----|:---:|----:
默认|左对齐|居中对齐|右对齐
A | B | C | D
```

> 表头一|表头二|表头三|表头四
> -----|:----|:---:|----:
> 默认|左对齐|居中对齐|右对齐
> A | B | C | D


代码块
====

代码块使用 4 个空格符或 1 个制表符开头。当然，更简单的方法是使用围栏式代码块，以```` ``` ````开始首行（可以在后面加语言代码），并在最后以```` ``` ````结束。代码内用到```` ``` ````字符时继续增加连续的`` ` ``字符。

````
```javascript
// 代码演示
class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }

  greet() {
    console.log(`Hello, I'm ${this.name}, ${this.age} years old.`);
  }
}

const person = new Person("Alice", 15);
person.greet();
```
````

> ```javascript
> // 代码演示
> class Person {
>   constructor(name, age) {
>     this.name = name;
>     this.age = age;
>   }
>
>   greet() {
>     console.log(`Hello, I'm ${this.name}, ${this.age} years old.`);
>   }
> }
>
> const person = new Person("Alice", 15);
> person.greet();
> ```


分隔线
====

使用 3 个以上的`-`、`_`、`*`建立分割线，中间可以插入空格，但是不要有其它的内容。

```
_____
```

> _____


行内代码
====

使用`` `代码` ``表示行内代码。代码内用到`` ` ``字符时使用多个连续`` ` ``包裹。

```
`foo`

`` foo ` bar ``
```

> `foo`
>
> `` foo ` bar ``


强调
====

有两种强调方式，使用`*文字*`或`_文字_`表示`<em>`强调，通常是斜体。使用`**文字**`或`__文字__`表示`<strong>`强调，通常是粗体。

注1：各种解析器在处理未断字的`_`和`*`时的处理方法各不相同，例如在本应用中，`a_b_c`显示为a_b_c，而`a*b*c`显示为a*b*c。

注2：部分解析器还支持删除符`~~文字~~`，配合前两种强调，正好可以分别用作文档修订的标记、插入和删除。这甚至可以替代任务列表的功能。

```
*斜体标记*，__粗体插入__，还有~~删除线~~

任务列表示例：

+ __2025-6-7__
+ ~~已完成的任务~~
+ ~~已完成的任务~~
+ *正在进行的任务*
+ 普通任务
+ 普通任务
```

> *斜体标记*，__粗体插入__，还有~~删除线~~
>
> 任务列表示例：
>
> + __2025-6-7__
> + ~~已完成的任务~~
> + ~~已完成的任务~~
> + *正在进行的任务*
> + 普通任务
> + 普通任务


链接
====

链接一般分为行内式和参考式两种。行内式链接的格式为`[文字](链接)`，一般是一次性使用。参考式链接的格式为`[文字][编号]`，然后在文章内（文章底部或段落结尾处都可以）用单独的段落定义这个`[编号]`代表的链接地址`[编号]: 链接`。参考式便于在文章内多次使用。

注：部分解析器支持将网址自动转换为链接，但复杂的网址是否能被正确转换还请自行确认。

```
文件链接：[Git 命令](git.md)

锚点链接：[强调](#强调)

网址链接：[淘宝][taobao] 也是 [淘宝网][taobao]

[taobao]: https://www.taobao.com/

自动转换链接： https://www.qq.com/ 或者 www.qq.com
```

> 文件链接：[Git 命令](git.md)
>
> 锚点链接：[强调](#强调)
>
> 网址链接：[淘宝][taobao] 也是 [淘宝网][taobao]
>
> [taobao]: https://www.taobao.com/
>
> 自动转换链接： https://www.qq.com/ 或者 www.qq.com


图片
====

图片与链接非常像，只是前面要加一个`!`，而且也分为行内式和参考式两种。行内式图片的格式为`![文字](图片地址)`，一般是一次性使用。参考式图片的格式为`![文字][编号]`，然后在文章内（文章底部或段落结尾处都可以）用单独的段落定义这个`[编号]`代表的图片地址`[编号]: 图片地址`。参考式便于在文章内多次使用。

注：本解析器可以在图片中使用锚点链接显示 [Lucide 图标](https://lucide.dev/icons/)，语法为`![text](#icon "css")`（这里使用的是 css 中定义的样式，所以直接使用颜色值无效）。如下示例：

```
![云下载](images/cloud-download.svg)

常用图标可以尝试参考式：

[info]: #message-square-more "blue"

```

反斜线
====

反斜线用于将后面的符号原文呈现。例如`*`原本是表示强调，但使用`\*`就可以呈现原始的`*`了。

注：HTML 字符实体（如`>`、`&`等）在 Markdown 中会自动转化，所以不需要使用反斜线。

```
\*原文呈现\*
```

> \*原文呈现\*


HTML
====

Markdown 语法兼容大部分 HTML 代码，直接添加即可。需要注意的是，HTML 行内元素内的 Markdown 语法会被正常处理，而块级元素内的 Markdown 语法不处理。

```
<font color="red">虽然是红色，但是可以*强调*</font>

<p>这里*强调无效*</p>
```

> <font color="red">虽然是红色，但是可以*强调*</font>
>
> <p>这里*强调无效*</p>
