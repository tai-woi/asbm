
ASBM 内部链接
====

ASBM 主要用作对 Markdown 文件的解析，它使用网址的查询字符串作为内部链接的地址，链接的根全部都指向唯一的 HTML 文件（单文件系统）。

如果以服务器默认访问的文件作为唯一的访问地址，则完整的网址形如：

```
https://domain/sitepath/?dir/filename.md#hash

# 文件名必备，内部链接路径 dir/ 可选，自动跳转锚点 #hash 可选
```

ASBM 内部链接即`?`及其以后的部分。它遵从 Markdown 文件的链接路径，完全不关心 HTML 文件的地址（不会动也不会改变）。例如对于上述网址的访问，实际需要解析的 Markdown 文件的完整地址为：

```
https://domain/sitepath/dir/filename.md
```

并在解析完成后自动跳转到`#hash`锚点。（可以看到，实际上就是去掉`?`后的地址）


Markdown 链接
====

Markdown 文件间的链接必须指明目标文件的文件名，可使用相对路径或绝对路径，也可指定锚点。ASBM 会自动将这些路径转换为内部链接格式。

例如，对于 Markdown 文件`dir/file1.md`（它的内部链接是`?dir/file1.md`），在 Markdown 文件间跳转时，其链接转换为：

+ `[文件2](file2.md)`转为`?dir/file2.md`
+ `[文件3](../file3.md)`或`[文件3](/file3.md)`转为`?file3.md`
+ `[文件4](subdir/file4.md)`转为`?dir/subdir/file4.md`

这种转换方式完全符合 Markdown 文件间的跳转方式。即使没有 ASBM，用户也可以非常直观的了解文件间的关系。

对于 Markdown 文件内的其它链接，如`http`、`ftp`、`mailto`等，直接使用即可，无需特别转换。

ASBM 将需要解析的 Markdown 文件转换为 HTML 格式后，会根据文件路径自动处理面包屑，根据文件内容自动生成标题纲要等，搭配本身的页面框架，就实现了方便查阅的 HTML 排版格式。


可能的错误
====

如果您在浏览页面时发生链接不规范的错误，这通常是由于：

+ 您直接访问的网址有误
+ 网站内容提供者编写 Markdown 时使用了不规范的网址
+ 网址查询字符串部分混入了不相干的数据

相关情况，可联络网站管理员。
