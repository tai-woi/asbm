
简介
====

ASBM 是 Any Server But Markdown 的缩写，目标是无论使用哪种网页服务器（Nginx, Apache, Caddy, Lighttpd, IIS，甚至开发工具 Python, PHP, VS Code, WebStorm, Xed-Editor 等自带的微服务器），用户都可以使用浏览器实时访问站内的 Markdown 文件，并且浏览器会自动将文件结构和文字转换为方便查阅的 HTML 排版格式。

ASBM 的基础是 HTML, JavaScript 和 CSS，但它实现了如下功能：

+ 单页面应用
+ 数据输出分离
+ 允许将 Markdown 文件作为区块加载到 HTML
+ 根据文件路径自动生成面包屑和网页标题
+ 根据 Markdown 文件内容自动生成标题纲要，并允许快捷跳转
+ __期待以后支持解析其它文本文件为 HTML（例如 CSV 转 HTML 表格）__
+ 错误记录和显示
+ 极度简捷

通过对 [Marked 解析器](http://marked.js.org/) 的扩展，它还支持：

+ [Lucide 图标](https://lucide.dev/icons/)
+ [Highlight.js 语法高亮](https://highlightjs.org/)
+ [Katex 公式](https://katex.org/docs/supported)
+ 根据您的需要自行扩展


应用
====

开发 ASBM 最初的目的是用作*个人效率工具*，如根据时间线产生的日记，需要分部门或分步骤处理的任务，零散和分类组织的记事，还有存放知识的知识库，软件编程方面的手册等。

但在实际使用过程中，我们发现针对性自定义和美化后，ASBM 几乎可以实现静态网站的绝大部分功能，如：

+ 网址导航
+ 博客记录
+ 文档手册
+ 图书城
+ 宣传展示

此外，它甚至可以作为开发工具自带的文档系统。具体要实现什么和怎么实现完全取决于您自己！


安全性
====

ASBM 除了享受静态网站的急速加载外，它无需任何后台代码，所以没有任何后端漏洞风险（除了服务器本身的风险），可以低成本托管到任意支持静态网页的站点。

另一方面，虽然 ASBM 本身不支持加密（完全开源，甚至对网站访问者都是如此），但实际加密的方式却有很多，包括：

+ 很多服务器本身具备加密功能，例如使用 htpasswd 工具
+ 使用简单的 Python, PHP 加密代码
+ 主机托管商的控制面板（如 cPanel）通常也具有目录隐私保护功能
+ 部分 CDN（如 CloudFlare）还可以进一步指定防火墙规则

具体加密方式请查阅您服务器的相关资料。


版本控制
====

ASBM 整个开发环境和写作文章都是纯文本，使用 Git 可以精准控制文档的修订历史。

而且用户使用 Git 时，还可以进一步利用 Git 的权限控制功能作为团队协作时的补充权限。这通常通过 Git 托管服务平台或自建 Git 服务器（如 Gitolite）时增加特定的配置来实现。
